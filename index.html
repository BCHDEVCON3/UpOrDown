Bitcoin Cash Defi

Using AnyHedge And Price Oracle

Eric is a degenerate gambler.

The Idea

The idea is to allow anyone to bet on the price of anything going up or down using Defi on Bitcoin Cash. This can be achieved using AnyHedge and the Price Oracle. It also uses Bitcoin.com REST APIs. Both AnyHedge and the Oracle are open source.

Depending on which side wins the bet, the Oracle will sign and publish a price message for AnyHedge to liquidate the contract in a deliberate way such that winning side of the contract wins the full amount (minus the commission fees). The winnings will be sent to the same wallet address from which the bet funds came from. Therefore, user accounts are not even required.

Given that I don't have sufficient time to complete this project for the Bitcoin Cash DevCon III, I find that it would make more sense to explain how it would work rather than try to code a complete demo. Hopefully, this will encourage more developers to start building Defi projects on Bitcoin Cash using AnyHedge and the Oracle. I am sure there are much more creative and capable developers out there than I am.

Additional Thoughts

This is an over-simplistic example of using AnyHedge + Oracle to create a Defi contract on Bitcoin Cash. There are many issues which need to be considered such as, what happens if there is a lack of liquidity?

The probability of something going up or down is usually not the same. Therefore, the rates may need to be adjusted accordingly.

The rate of price changes (volatility) differ between different types of bets. Therefore, parameters such as the liquidation date may need to factor these into account.

The opening and closing time of bets will need to be adjusted to avoid cheating.

These issues are beyond the scope of this hackathon project. Feel free to take everything and build your own Bitcoin Cash Defi projects with AnyHedge.
